# API 비용 분석 플랫폼 - PRD (Product Requirements Document)

## 개요
**제품명**: API 비용 분석 플랫폼  
**대상 시장**: SaaS CTO, 엔지니어링 리더, 스타트업 운영팀  
**카테고리**: Cost Management / FinOps  
**기술 스택**: Next.js 15 + shadcn/ui + Drizzle ORM + SQLite

---

## 1. 문제 정의

### 통증점 (Pain Statement)
- SaaS 회사가 10개 이상의 API를 사용하면서 각 API별 비용, 사용량, 낭비를 추적하기 어려움
- 월간 서프라이즈 청구서로 인한 예산 외 지출
- 미사용 API와 불필요한 호출 감지 불가능
- 팀별 비용 할당 및 책임 추적의 어려움

### 증거
- 20개 이상 API 사용 스타트업: API별 비용 추적 어려움 (SaaS Spend Management Survey 2026)
- 기술 회사의 38%가 API 예산의 20-40%를 미사용 또는 중복 서비스에 낭비 (Cloud Cost Intelligence Report 2026)

## 2. 페르소나 (Target Personas)

### 페르소나 1: 김준호 (CTO, 30인 스타트업)
- **역할**: 엔지니어링 리더, 인프라 및 비용 최적화 담당
- **목표**: API 비용을 투명하게 파악하고 낭비를 줄이기
- **고통점**: Stripe, OpenAI, Twilio, AWS API 등 여러 제공자의 청구서를 각각 관리하는 번거로움
- **니즈**: 한눈에 보는 대시보드, 자동 비용 감지, 팀별 비용 할당

### 페르소나 2: 이미경 (엔지니어링 관리자, 150인 회사)
- **역할**: 기술 팀 관리, 예산 결정
- **목표**: 엔지니어 팀의 API 지출을 모니터링하고 최적화 제안
- **고통점**: 여러 API의 사용량과 비용 예측이 어려움
- **니즈**: 프로젝트별/팀별 비용 분배, 이상 징후 알림, 비용 예측

### 페르소나 3: 박운영 (스타트업 운영팀)
- **역할**: 재무, 예산 관리
- **목표**: 월간 지출 예산 추적 및 비용 절감 기회 발굴
- **고통점**: API 청구서가 분산되어 있고, 어느 서비스가 낭비되는지 불명확함
- **니즈**: 비용 요약, 트렌드 분석, 비용 절감 기회 식별

## 3. MoSCoW 우선순위 (Must-Have 기능 8개+)

### MUST-HAVE (필수)
1. **자동 API 비용 수집 및 분석 대시보드** (Killer Feature)
   - 주요 API 제공자별 비용 데이터 통합 (Stripe, OpenAI, Twilio, AWS API, SendGrid 등)
   - 월간 / 분기별 비용 요약 그래프
   - 실시간 업데이트 (최소 일일 1회)

2. **API별 사용량 추적 및 비용 예측**
   - API별 월간 비용 기록
   - 추세 분석 (지난 3개월 평균 대비)
   - 다음 월 비용 예측

3. **미사용 API 및 불필요한 호출 자동 감지** (Core Feature)
   - 0 사용량 API 자동 표시
   - 비정상적인 비용 증가 감지
   - 예상 범위를 초과한 비용 알림

4. **팀별/프로젝트별 비용 분배**
   - 비용 태그 관리 (팀/프로젝트별 라벨링)
   - 팀별 월간 비용 합계 및 차트
   - 팀별 상세 비용 분석

5. **비용 이상 감지 및 실시간 알림**
   - 임계값 설정 기능
   - 임계값 초과 시 알림 (이메일 / 인앱)
   - 비용 급증 감지

6. **검색 및 필터링**
   - API 이름으로 검색
   - 비용 범위 필터
   - 날짜 범위 필터
   - 팀/프로젝트별 필터

7. **대시보드 및 통계**
   - 총 API 비용 요약 (월간/분기별)
   - 비용 상위 5개 API 시각화
   - 월간 비용 추세 그래프
   - 팀별 비용 분포

8. **빈 상태 / 로딩 / 에러 상태 처리**
   - API 데이터 없을 때 안내 메시지
   - 데이터 로딩 중 스켈레톤 / 스피너
   - 에러 발생 시 명확한 에러 메시지

### SHOULD-HAVE (권장)
- 월간 비용 리포트 자동 생성
- CSV/PDF 내보내기
- Slack 알림 통합
- 비용 절감 권장사항 AI 생성

### COULD-HAVE (선택)
- 다중 사용자 역할 관리
- API 연동 자동화 (Stripe API 직접 연결)
- 예산 대비 실제 지출 추적

### WON'T-HAVE (제외)
- 결제 처리 기능
- API 사용량 제어 기능

## 4. 화면 구조 (6개+ 화면)

### 4.1 대시보드 (Dashboard)
- **목적**: 한눈에 전체 API 비용 파악
- **구성**:
  - 월간 총 비용 (큰 숫자)
  - 월간 비용 변화 (지난달 대비 % 증감)
  - 비용 상위 5개 API (바 차트)
  - 월간 비용 추세 (라인 차트)
  - 팀별 비용 분포 (파이 차트)
  - 최근 이상 감지 알림 (리스트)

### 4.2 API 목록 (API List)
- **목적**: 모든 API의 비용을 표로 확인
- **구성**:
  - API 이름, 월간 비용, 월간 사용량, 상태 (활성/비활성)
  - 검색 및 필터 (이름, 비용 범위, 상태)
  - 정렬 (비용 높은 순, 사용량 높은 순)
  - 각 행을 클릭하여 상세 정보로 이동

### 4.3 API 상세 (API Detail)
- **목적**: 특정 API의 상세 분석
- **구성**:
  - API 이름, 제공자, 비용 기록 (3개월)
  - 월별 비용 그래프
  - 월별 사용량 (요청 수 등)
  - 다음 월 예측 비용
  - 팀별 비용 분배
  - 태그 관리 (팀/프로젝트 라벨)
  - 비용 이상 알림 설정

### 4.4 팀별 분석 (Team Analysis)
- **목적**: 팀/프로젝트별 비용 추적
- **구성**:
  - 팀별 월간 비용 리스트
  - 팀별 비용 파이 차트
  - 각 팀이 사용하는 API 목록
  - 팀별 상세 조회

### 4.5 알림 설정 (Alert Settings)
- **목적**: 비용 이상 감지 및 알림 설정
- **구성**:
  - 임계값 설정 (월간 총 비용, API별 비용)
  - 알림 채널 설정 (이메일, 인앱)
  - 알림 빈도 설정 (즉시, 일일, 주간)
  - 알림 이력 조회

### 4.6 설정 (Settings)
- **목적**: 애플리케이션 설정
- **구성**:
  - 사용자 프로필 (이름, 이메일)
  - 통화 선택 (USD, EUR, KRW 등)
  - API 키 관리 (연동할 API 인증 정보)
  - 알림 기본 설정

### 추가 화면

### 4.7 빈 상태 (Empty State)
- 첫 API 추가 가이드
- CTA 버튼 (API 추가)

### 4.8 로딩 상태 (Loading State)
- 스켈레톤 로더
- 진행률 표시

## 5. 데이터 모델 (Schema)

### Users
```sql
id, email, name, createdAt, updatedAt
```

### APIs
```sql
id, userId, name, provider, monthlyBudget, createdAt, updatedAt
```

### APICosts
```sql
id, apiId, month (YYYY-MM), cost (float), usage (int), createdAt
```

### Teams
```sql
id, userId, name, createdAt, updatedAt
```

### TeamAPIs (Many-to-Many)
```sql
id, teamId, apiId, allocatedBudget, createdAt
```

### Alerts
```sql
id, userId, type (threshold), threshold, channel (email/in-app), isActive, createdAt
```

### AlertHistories
```sql
id, alertId, triggerDate, message, status (sent/pending)
```

## 6. 경쟁분석 & 차별점

### 경쟁자
| 경쟁자 | 강점 | 약점 |
|--------|------|------|
| **Finout** | 엔터프라이즈급 기능 | $5K+/월 높은 가격, 복잡한 설정 |
| **CloudHealth** | AWS 통합 | AWS 중심, 다른 API 제공자 부족 |
| **일반 청구서 분석** | 저가 | API별 상세 분석 부족 |

### 차별점
1. **API 특화**: Stripe, OpenAI, Twilio 등 주요 API 제공자 직접 통합
2. **자동 감지**: 미사용 API 및 비용 이상 자동 감지
3. **저가 모델**: 월 $99 (스타트업 대상)
4. **간단한 설정**: 5분 내 시작 가능
5. **한국어 UI**: 모든 인터페이스 한국어 제공

## 7. UI/UX 가이드 (한국어)

### 언어 설정
- **모든 UI는 한국어**: 버튼, 레이블, 플레이스홀더, 메시지, 메뉴
- **숫자 포맷**: 1,000,000 (한국식 쉼표)
- **통화**: ₩, $, € 등 사용자 선택
- **날짜**: YYYY년 MM월 DD일 형식

### 컬러 팔레트
- **Primary**: #3B82F6 (파랑)
- **Success**: #10B981 (초록)
- **Warning**: #F59E0B (주황)
- **Danger**: #EF4444 (빨강)
- **Neutral**: #6B7280 (회색)

### 타이포그래피
- **Heading 1**: 28px, 700 weight
- **Heading 2**: 20px, 600 weight
- **Body**: 14px, 400 weight
- **Caption**: 12px, 400 weight

### 반응형 디자인
- **Desktop**: 1024px 이상
- **Tablet**: 768px ~ 1023px
- **Mobile**: 320px ~ 767px
